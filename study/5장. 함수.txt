1.3 지역변수(local var)와 전역변수(global var)

- 지역변수와 전역변수
	- 변수의 값을 조회할 때, 로컬 심볼 테이블 - 전역 심볼 테이블 - 내장 이름 테이블 순서로 조회
	- 렉시컬 특성 : 변수의 선언 위치에 따라 변수 범위가 정해짐
		-> 전역변수로 값이 잡혀있더라도, 
		    함수 내에서 지역변수로 새로 지정되면 전역변수와 관련 없이 따로 할당이 필요함
	- global을 활용하여 함수 내에서 전역변수 사용 가능
	ex)
	g_var = 100 #전역변수

	def func3():
		global g_var # func3 함수 내의 g_var는 전역변수를 사용함을 선언
		print("함수 안 :", g_var) #전역변수
		g_var = 200 #전역변수
	func3()
	print("함수 밖 :", g_var)

3. 함수의 매개변수

3.1 기본값을 갖는 매개변수

- 기본값을 갖는 매개변수 : 입력이 없을 때  기본값을 가진다
	- 순서인수가 있을 때, 기본값을 갖는 매개변수의 뒷순서로 갈 경우 에러 발생

	ex)
	def make_url (port=80, ip): 
		return f"http://{ip}:{port}" >>> error

	def make_url (ip, port=80): #ip:순서인수, port:기본값을 갖는 매개변수
		return f"http://{ip}:{port}"

	print(make_url('www.naver.com')) >>> http://www.naver.com:80
	print(make_url(port=9090, ip='www.naver.com')) ; 키워드 인수

3.2 기본변수를 갖는 매개변수

- 기본변수가 스칼라 변수(문자, 숫자, 논리형)일 때
	- 기본값은 함수가 정의되는 시점 기준

	ex)
	i = 5
	def func2(arg = i):
		print('arg =', arg)

	i = 10
	func2() >>> arg = 5

- 기본변수가 리스트, 튜플, 셋, 딕셔너리 또는 객체일 때
	- 함수가 호출되는 시점의 item이 기본값

	ex)
	lst = []
	def func3(a, L=lst):
		L.append(a)
		return L

	print("lst :", lst) >>> lst : [3]

	print("lst :", lst) >>> lst : [3, 3]

3.4 튜플 매개변수를 이용한 가변인수 설정

- 함수 정의 ex1)
	def [함수명](a, b, c, ...):
		return 반환 값

- 함수 정의 ex2)
	# args앞의 * : 매개변수들(인수들)이 args 튜플로 전달
	def add(*args):
	"전달받은 매개변수들의 합을 return. 매개변수가 없을 경우 0을 return"
		tot = 0 # 누적변수
    		for num in args:
       			tot += num # tot = tot + num	
		return tot
		
- 튜플 매개변수와 일반 매개변수가 같이 있을 때
	ex)
	def concat(*subject, sep):
		return sep.join(subject)
	concat('python', 'web', 'Oracle', 'MySQL', 'DL', 'ML', 'LLM', 'flask/django', ' - ') >>> error

- 해결책1 : 호출시 파라미터 이름을 지정
	concat('python', 'web', 'Oracle', 'MySQL', 'DL', 'ML', 'LLM', 'flask/django', sep=' - ')
	>>> 'python - web - Oracle - MySQL - DL - ML - LLM - flask/django'

- 해결책2 : 기본값을 갖는 매개변수 활용
	def concat(*subject, sep=" "):
		return sep.join(subject)
	concat('python', 'web', 'Oracle', 'MySQL', 'DL', 'ML', 'LLM', 'flask/django', sep=' - ')
	>>> 'python - web - Oracle - MySQL - DL - ML - LLM - flask/django'

- 해결책3 : sep을 순서인수로 맨앞에 정의
	def concat(sep, *subject):
		return sep.join(subject)
	concat('#', "국어", "영어", "수학") >>> '국어#영어#수학'

3.5 딕셔너리 매개변수
- 여러 개의 키워드인수를 딕셔너리 형태로 받아서 가변 키워드 인수로 처리

	ex)
	def func(**args):
		print(args)
		print(type(args))

			print("{}:{}".format(key, value))

		print('point :', args['point'])

	func(color="red", width=2, point="O", name='선그래프')
	>>>	{'color': 'red', 'width': 2, 'point': 'O', 'name': '선그래프'}
		<class 'dict'>
		color:red
		width:2
		point:O
		name:선그래프
		color : red
		width : 2
		point : O

3.6 함수 정의시 매개변수의 순서
- 순서인수 > 튜플인수(*) > 키워드인수(기본값) > 딕셔너리인수(**)

	ex)
	def func2(a, b, c, *d, e=10, **f):
		print("순서인수 a =", a)
		print("순서인수 b =", b)
		print("순서인수 c =", c)
		print("튜플인수 d =", d)
		print("키워드인수 e =", e)
		print("딕셔너리인수 f =", f)

	func2('a', 'b', 'c', 1, 2, 3, e=20, shape="rect", width=20, height=10)
	>>> 	순서인수 a = a
		순서인수 b = b
		순서인수 c = c
		튜플인수 d = (1, 2, 3)
		키워드인수 e = 20
		딕셔너리인수 f = {'shape': 'rect', 'width': 20, 'height': 10}

3.7 인수의 언패킹

- 튜플인수 언패킹
	- 패킹 : 함수 정의 시 매개변수 앞 * (튜플 매개변수)
	- 언패킹 : 함수 호출 시 매개변수 앞 * (리스트 / 튜플 >>> 각 요소별 매개변수)

- 딕셔너리 인수 언패킹
	- 패킹 : 함수 정의 시 매개변수 앞 ** (딕셔너리 매개변수)
	- 언패킹 : 함수 호출 시 매개변수 앞 ** (리스트 / 튜플 >>> 각 요소별 매개변수)
