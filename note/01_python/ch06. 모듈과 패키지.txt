ch06. 모듈과 패키지

- 1. 파이썬 표준 모듈 사용
	* 모듈(Module) : 함수, 변수, 클래스 등을 담고 있는 하나의 파이썬 코드 파일(*.py)
	* 패키지(Package) : 여러 개의 모듈(*.py)이 모여 있는 폴더 (폴더 내 __init__.py 파일 포함 권장)

	* 모듈 및 구성 요소 가져오기 (import 문법)
		- import A                 : A 모듈 로드 (사용 시 A.함수명() 구조 필수)
		- import A as B            : A 모듈을 B라는 별칭(Alias)으로 로드 (B.함수명()으로 호출)
		- from A import B          : A 모듈 내의 구성 요소 B(함수, 클래스, 변수)를 직접 로드 (A 없이 B()로 호출 가능)
		- from A import B as C     : A 모듈 내의 B 요소를 C라는 별칭으로 로드

	* 패키지 내 모듈 가져오기 (import 문법)
		- from A import B          : A 패키지 내의 B 모듈(또는 하위 패키지) 로드
		- from A.B import C        : A 패키지 안의 B 모듈에서 구성 요소 C(함수/클래스/변수) 로드
		- from A import B as C     : A 패키지 내의 B 모듈을 C라는 별칭으로 로드
		- from A.B import C as D   : A 패키지 내 B 모듈의 구성 요소 C를 D라는 별칭으로 로드

- 2. 사용자 정의 모듈
	* 목적 : 직접 작성한 함수, 클래스, 변수 등을 별도의 *.py 파일로 저장하여 필요할 때 호출 및 재사용
	* 모듈 탐색 및 가져오기 순서 :
		1) 메모리에 이미 로드된 모듈 확인
		2) 파이썬 내장 표준 모듈 라이브러리 디렉터리에서 검색
		3) sys.path 변수(리스트 형태)에 등록된 경로 순서대로 검색

- 3. 패키지 특징
	* 구조 : 모듈 파일들을 모아 관리하는 디렉터리 (과거 버전 패키지 인식을 위해 __init__.py 파일 포함 권장)
	* 특징 : 상위 패키지를 import 했다고 해서 하위 패키지나 모듈이 자동으로 import되지는 않음 (필요한 모듈을 하위 경로까지 정확히 지정하여 import 해야 함)

- 4. 패키지 설치 및 관리
	* 실행 환경 : 아나콘다 프롬프트(Anaconda Prompt) 또는 터미널 환경
	* 패키지 설치 :
		- pip install 패키지명[==버전]
		- conda install 패키지명[=버전]
		- uv pip install 패키지명[==버전] (고속 pip 대안 도구)
	* 설치 패키지 확인 :
		- pip show 패키지명       : 특정 패키지의 상세 정보 및 버전 확인
		- conda list [패키지명]   : 설치된 전체 패키지 목록 검색 (특정 패키지명 입력 시 해당 패키지 정보만 출력)
	* 패키지 삭제 :
		- pip uninstall 패키지명
		- conda remove 패키지명
