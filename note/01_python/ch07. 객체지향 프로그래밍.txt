ch07. 객체지향 프로그래밍

- 1. 객체와 클래스
	* 클래스 : 객체를 만들기 위한 틀 (추상적인 디자인 공간)
		- 파이썬의 모든 클래스는 기본적으로 최상위 object 클래스를 자동 상속받음
	* 객체(Instance) : 클래스를 바탕으로 메모리에 실제 구현된 데이터(실체)

- 2. 생성자와 소멸자
	* 생성자 __init__(self, [매개변수], ...) : 객체가 생성될 때 자동 실행 (인스턴스 변수 초기화, 미정의 시 기본 생성자 자동 호출)
	* 소멸자 __del__(self) : 객체가 소멸될 때 자동 실행 (인스턴스 객체의 레퍼런스 카운트가 0이 될 때 호출)
	* self : 현재 함수(메서드)를 호출한 객체(인스턴스) 자기 자신을 가리키는 대명사 (PEP 8 권장 약속)

- 3. 변수와 메소드(함수)
	* 변수의 종류
		- 인스턴스 변수(멤버 변수) : 각 인스턴스마다 독립적으로 가지는 변수 (self.변수명 형태로 사용)
		- 클래스 변수 : 클래스 내부(메서드 밖)에 선언되며, 모든 인스턴스가 공유하는 변수
			- [클래스명].[클래스변수] 수정 시 : 모든 인스턴스에 변경 사항이 공유/반영됨
			- [객체명].[클래스변수] 수정 시 : 해당 객체 전용 인스턴스 변수가 새로 동적 생성되어 독립 처리됨 (기존 클래스 변수와 단절)
		- 비공개 변수 : 변수명 앞에 언더바 2개(__)를 붙여 외부 직접 접근을 제한하는 기능 (네임 마글링 적용)
	* 메소드의 종류
		- 인스턴스 메소드 : self를 첫 번째 매개변수로 받으며, 인스턴스 속성(멤버 변수)에 직접 접근하여 연산 수행
		- 클래스 메소드 : @classmethod 데코레이터 사용, cls를 첫 번째 매개변수로 받아 클래스 속성에 접근
		- 스태틱 메소드 : @staticmethod 데코레이터 사용, self나 cls를 받지 않으며 클래스/인스턴스 속성에 직접 관여하지 않는 독립적인 유틸리티 함수

- 4. 정의 및 호출 문법
	class Class_name(상속_받을_클래스명):
		'''클래스 설명 독스트링'''
		클래스변수 = 값

		# 생성자
		def __init__(self, [멤버변수]=[기본값], ...):
			self.멤버변수 = 멤버변수

		# 소멸자
		def __del__(self):
			객체 소멸 시 수행할 명령어

		# 인스턴스 메소드
		def method_name(self, [매개변수], ...):
			return self.멤버변수

		# 클래스 메소드
		@classmethod
		def class_method_name(cls, [매개변수], ...):
			return cls.클래스변수

		# 스태틱 메소드
		@staticmethod
		def static_method_name([매개변수], ...):
			독립적으로 수행할 명령어

	# 인스턴스 생성 및 호출 예시
	var_name = Class_name()
	var_name.method_name()               # 인스턴스 메소드 호출 (객체를 통해 호출)
	Class_name.class_method_name()       # 클래스 메소드 호출 (클래스명으로 직접 호출)
	Class_name.static_method_name()      # 스태틱 메소드 호출 (클래스명으로 직접 호출)
	var_name.__dict__                    # 객체의 멤버 변수와 값을 딕셔너리 형태로 반환

- 5. 상속과 재정의 (Inheritance & Override)
	* 상속 : 기존 클래스의 속성과 메소드를 그대로 물려받아 재사용하는 방법
		- 정의 문법 : class 자식클래스명(부모클래스명):
	* 메소드 오버라이딩 (재정의) : 부모로부터 물려받은 메소드를 자식 클래스 목적에 맞게 새로 재정의하여 사용하는 것
	* super() : 부모 클래스를 지칭하는 키워드 (부모 클래스의 생성자나 메소드를 호출할 때 사용, 예: super().__init__())
