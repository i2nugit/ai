8장. 예외처리

- 예외처리 정의 : 프로그램 실행 중 발생하는 예기치 않은 사건(Error)에 대비하는 것

- 기본 문법
	try:
		예외가 발생할 가능성이 있는 명령어 블록
	except 예외타입1:
		해당 예외1이 발생했을 때 실행할 명령어
	except 예외타입2 as e:
		해당 예외2가 발생했을 때 에러 메시지(e)를 출력하거나 처리할 명령어
	except Exception as e:
		[최상위 예외] 지정되지 않은 나머지 모든 예외를 처리 (반드시 가장 아래에 위치)
	else:
		try 블록 내에서 예외가 전혀 발생하지 않았을 때만 실행할 명령어 (자주 사용 안 함)
	finally:
		예외 발생 여부와 상관없이 무조건 마지막에 실행할 명령어 (자원 반납 등)

- 예외 처리 주의사항
	* 다중 except 사용 시, 상위 예외 타입(상속 관계에서 부모 클래스)은 반드시 하위 예외 타입의 '아래'에 위치해야 함 (위에서부터 순차 매칭되기 때문)

- 강제 예외 발생 (raise)
	* 문법 : raise 예외타입("에러메시지")
	* 활용 : 
		- 중첩 루프 탈출 : 안쪽 루프에서 raise 후 루프 바깥의 except 블록에서 받아 한 번에 탈출
		- 추상 메서드 구현 구현 강제 : 부모 메서드 내부에 raise를 두어 자식의 오버라이딩을 강제

- 추상 클래스 (Abstract Class)
	* 정의 : 추상 메서드가 하나 이상 포함되어, 직접 객체(인스턴스)를 생성할 수 없는 클래스
	* 목적 : 자식 클래스들이 특정 메서드를 반드시 구현(재정의; 오버라이딩)하도록 구조를 강제함
	* 구현 표준 문법 :
		from abc import ABC, abstractmethod
		class 부모클래스(ABC):
			@abstractmethod
			def 메서드명(self): pass 

- 자동 자원 관리 (with문)
	* 특징 : 블록 내부 코드가 완료되거나 중간에 에러가 발생하더라도, 블록을 벗어나는 순간 자동으로 f.close() 같은 자원 해제 메서드를 실행해 줌
