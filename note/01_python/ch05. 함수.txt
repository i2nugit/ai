5장. 함수

- 1. 함수의 정의 및 사용
	* 정의 문법 : def 함수명(매개변수, ...):
	* 특징 : 파이썬은 '오버로딩(Overloading)'을 지원하지 않음 (매개변수 형태만 다른 같은 이름의 함수 정의 불가, 마지막에 정의된 함수가 이전 함수를 덮어씀)
	* 독스트링(docstring) : 함수 선언문 바로 아래 삼중 따옴표(''' 또는 """)로 작성하는 함수 설명서
	* 변수 참조 우선순위 (LEGB Rule) :
		- 함수 내부 -> Global(전역 변수) -> 내장 함수/이름 영역 순서로 조회
		- global 키워드 : 함수 내부(Local)에서 전역 변수(Global)의 값을 직접 수정하고자 할 때 선언
	* 인수 전달 방식 :
		- 값에 의한 호출처럼 동작 : 불변 객체 (숫자, 문자열, 논리형, 튜플) -> 함수 내에서 변경해도 원본 유지
		- 참조에 의한 호출처럼 동작 : 가변 객체 (리스트, 셋, 딕셔너리, 사용자 정의 객체) -> 함수 내에서 변경 시 원본도 함께 변경됨
	* 함수의 이름 변경 : 파이썬은 함수를 일반 변수처럼 다루며, 다른 변수에 대입하여 이름 변경 가능

- 2. return
	* 특징 : 함수의 실행 결과를 호출한 위치로 전달하고 함수를 즉시 종료함
	* 다중 값 반환 : return a, b, c 처럼 여러 값을 반환할 시, 파이썬은 이를 하나의 '튜플(tuple)'로 묶어서 반환함

- 3. 함수의 매개변수
	* 기본값 매개변수 사용 시 주의점 :
		- 스칼라 변수 : 기본값은 '함수가 정의되는 시점' 기준으로 단 한 번만 생성 및 고정됨
		- 가변 객체 (리스트, 딕셔너리, 셋 등) : 기본값으로 빈 리스트(예: [] ) 등을 사용하면 모든 함수 호출 시 동일한 객체가 공유되어 값이 계속 누적되는 심각한 오류 발생 
									(기본값으로 None을 주고 함수 내부에서 새로 생성하는 방식을 권장)
	* 위치 인수 : 함수 정의에 지정된 순서대로 인수를 전달하는 방식
	* 키워드 인수 : 인수 이름을 명시하여 순서에 상관없이 값을 전달하는 방식 (예: func(age=20, name="Kim"))
	* 가변 위치 인수 (*args) : 개수가 정해지지 않은 복수의 인수를 '튜플(tuple)' 형태로 한 번에 받음
	* 가변 키워드 인수 (**kwargs) : 개수가 정해지지 않은 복수의 키워드 인수를 '딕셔너리(dict)' 형태로 한 번에 받음
	* 매개변수 지정 순서 규칙 (오류 방지 필수 순서) :
		- 위치 인수 >> 가변 위치 인수(*args) >> 키워드 인수(기본값 매개변수) >> 가변 키워드 인수(**kwargs)
	* 인수 언패킹 : 
		- 함수 호출 시 튜플/리스트 앞에 * 기호를 붙이면 원소들이 각각의 위치 인수로 풀어져서 전달됨
		- 함수 호출 시 딕셔너리 앞에 ** 기호를 붙이면 키-값 쌍들이 각각의 키워드 인수로 풀어져서 전달됨

- 4. 람다식 Lambda
	* 정의 문법 : lambda [매개변수]: [리턴값] (이름이 없는 한 줄짜리 축약형 함수)
	* 용도 : 다른 함수의 매개변수로 전달되거나, 함수 자체를 반환할 때 주로 사용
	* 대표적인 연계 활용 함수 :
		- map(func, vector_data)    : 벡터 데이터(리스트, 튜플 등 선형 데이터)의 각 원소에 함수를 일괄 적용하여 매핑된 결과 반환
		- filter(func, vector_data) : 함수의 리턴값이 True인 원소만 필터링하여 남김
		- sort(key=func, reverse=T/F) : key 매개변수에 정렬 기준 값을 반환하는 람다/함수를 지정하여 맞춤형 정렬 수행

- 5. 파이썬 내장함수
	* 특징 : 별도의 import 문 없이 파이썬 실행 환경에서 언제든지 직접 호출하여 사용 가능한 내장 함수군 (예: print(), len(), sum(), type(), id() 등)
